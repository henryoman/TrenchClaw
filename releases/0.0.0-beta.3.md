### Improve chat stream robustness and sanitize UI input (`aabeb7f`)

- Commit: `aabeb7f7b69bae3e6efc9a29b299eeaaad70ae81`
- Category: Other
- Author: Henryoman
- Date: 2026-03-18
- Body:
```text
Make the runtime and frontend more resilient to malformed UI input and tool-stream errors. Changes include:

- Frontend: add a /v1 proxy in vite config.
- AI gateway: increase maxOutputTokens for operator/workspace lanes and set a smaller limit for background-summary.
- GUI transport parsing: sanitize chat message parts to only accept trimmed text parts, ignoring malformed or non-text parts.
- Runtime chat service: add helpers to identify text chunk IDs and recoverable tool-only stream errors; track and suppress pre-tool text parts, use Promise.allSettled to handle stream/consume results, and reconstruct a recovered first-pass message list for fallback generation when needed.
- Tests: add coverage for sanitizing non-text parts and for ignoring suppressed pre-tool text chunks during fallback recovery.
- Remove an unused devnet airdrop export.

These changes allow the system to tolerate malformed inputs, avoid leaking suppressed pre-tool narration into later passes, and recover gracefully from certain streaming errors.
```
