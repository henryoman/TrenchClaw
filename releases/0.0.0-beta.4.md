# TrenchClaw v0.0.0-beta.4

`0.0.0-beta.4` is a tighter, cleaner beta focused on shipping what is actually working now, with a better default AI path for real operators.

The headline change in this release is the move to `openai/gpt-5.4-nano` as the default model. We previously leaned on a Hunter Alpha free path, but that stopped being a good public default once it became clear the model path was effectively cloaked, unstable for long-term product expectations, and no longer something we should present as a dependable free baseline. `GPT-5.4 Nano` gives TrenchClaw a much better default profile for beta users: low cost, large context, predictable availability, and a cleaner story for the shipped app.

## Highlights

- Switched the default AI model to `openai/gpt-5.4-nano`.
- Updated the product messaging and setup docs so the public install flow points users at the actual current default model.
- Removed Jupiter Trigger features from the current beta surface and cleaned up the related docs and shipped messaging.
- Kept the Bun-first tooling and release path aligned with the current product surface.

## Why The Model Switch Matters

- Better operator default: `GPT-5.4 Nano` is the cheaper, practical high-context choice for a beta runtime that still needs room for long prompts, tool descriptions, and multi-step chat flows.
- More predictable shipping story: the old Hunter path was not a stable public-facing default anymore.
- Cleaner support burden: docs, settings, runtime defaults, and user expectations are now aligned again.

## Beta Scope Reminder

This beta is still about the narrow shipped-now surface:

- chat-driven local runtime workflows
- managed wallet reads and wallet organization
- Dexscreener research
- Jupiter Ultra swap workflows

Everything else should continue to be treated conservatively until it is fully proven in the shipped app.
