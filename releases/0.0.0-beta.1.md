# TrenchClaw 0.0.0-beta.1

`0.0.0-beta.1` is the first public beta-style release message for TrenchClaw.

This release is experimental. We do not recommend broad usage yet, and we strongly recommend waiting until `v0.1.0` if you do not specifically want to test early software.

## What Ships In This Release

- local `trenchclaw` install and launcher
- instance creation and sign-in
- vault and API key setup
- wallet creation and wallet lookup
- wallet backup/export
- direct transfers as a narrower beta surface
- Jupiter Ultra swap flows
- Solana data-fetching and market research flows
- chat-driven runtime/operator workflows
- Solana CLI and Helius CLI setup guidance for workflows that explicitly require them

## Setup Requirements

- Helius API key
- Jupiter Ultra API key
- AI provider key if you want chat-driven workflows
- Solana CLI only for workflows that explicitly require it
- Helius CLI only for workflows that explicitly require it

## Beta Boundaries

This release should be described as a narrow beta. The public message should focus on wallet operations, managed wallet reads, Dexscreener research, Jupiter Ultra, and chat-driven local workflows.

The following surfaces may exist in the repo or internal docs, but they are not the core promise of this release:

- trigger-based automation
- scheduled swaps and broader routine systems
- privacy flows
- devnet airdrop as a headline feature
- swap history as a headline feature
- non-Ultra public swap paths
- broad autonomous strategy claims

## Current Limitations

- many execution flows are still chat-driven
- there is no full dedicated GUI flow for every wallet operation
- advanced automation surfaces should be treated as out of scope for this release
- the product remains experimental and should be used carefully
