# Release Process

TrenchClaw has one public shipping path: GitHub Releases publishing standalone `trenchclaw` binaries.

## What Gets Published

The release workflow publishes these artifacts:

- `trenchclaw-<version>-darwin-arm64.tar.gz`
- `trenchclaw-<version>-linux-x64.tar.gz`
- `trenchclaw-<version>-linux-arm64.tar.gz`
- matching `.sha256` files
- matching `*.release-metadata.json` files

Each tarball contains only:

```text
trenchclaw
gui/
core/
release-metadata.json
```

No mutable state is bundled. Databases, logs, vaults, key files, and runtime state stay in `~/.trenchclaw` on the user machine.

## Workflow

Release publishing is handled by [`.github/workflows/release.yml`](/Volumes/T9/cursor/TrenchClaw/.github/workflows/release.yml).

Trigger:

- `workflow_dispatch`

Inputs:

- `version`
- `prerelease`

Release/version gate:

- `version` must exactly equal `v${package.json version}`
- `prerelease` must match whether `package.json version` is a beta (`-beta.N`)
- the workflow fails if the Git tag already exists

Job behavior:

1. `validate` first runs `bun run release:validate -- --version ... --prerelease ...`.
2. `validate` then runs `bun run lint`, `bun run typecheck`, and `bun run test`.
3. `build_release` runs once per target platform.
4. Each matrix job builds readonly assets with `bun run app:build`.
5. Each matrix job packages one tarball with `bun run release:package`.
6. Each matrix job smoke-tests the packaged tarball with `bun run release:smoke -- --artifact-path ...`.
7. `publish` downloads all packaged artifacts and creates the GitHub Release.
8. The GitHub Release uses `generate_release_notes: true`, so notes are generated by GitHub from the changes since the previous release.

Release note categories are configured by [`.github/release.yml`](/Volumes/T9/cursor/TrenchClaw/.github/release.yml).

## Local Release Prep

For local verification before dispatching the workflow:

```bash
bun run app:clean
bun run app:build
bun run bundle:verify
bun run release:package -- --version vX.Y.Z
bun run release:smoke -- --artifact-path dist/release/trenchclaw-vX.Y.Z-linux-x64.tar.gz
```
